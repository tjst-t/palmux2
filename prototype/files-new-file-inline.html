When the user clicks + New file, a placeholder row is inserted at the bottom of the current directory's listing (above the CTAs).

The input is autofocused with a placeholder hinting the kind of name expected. Both CTAs go disabled while the inline editor is active so the user can't open a second editor.

Keyboard contract:

  • Enter → create the file (or directory) with the typed name; opens it in the editor pane on success
  • Esc → discard the inline row, return to the normal list
  • / in the name → user can include subpaths (foo/bar.txt); parent dirs auto-created

Validation: empty name + Esc cancels; existing names show inline error in the hint line below the input (red, replaces the help text). Server returns 409 on dup.
