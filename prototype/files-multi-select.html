3 items selected via Cmd / Ctrl-click. No checkboxes — the leading 📁 / 📄 icon stays put on every row, so file ↔ folder distinction never depends on selection state. Selected rows get the accent-tinted background + 2px accent left border (same treatment Finder / VSCode use).

Selection rules — desktop:

  • Click → select single (replaces previous selection)
  • Cmd/Ctrl + click → toggle individual rows
  • Shift + click → range from last anchor
  • Esc or ✕ Cancel → clear selection
  • Delete / Backspace → invoke batch delete

Selection rules — touch:

  • Long-press a row → enter select mode, that row gets selected
  • While in select mode, a tap toggles the row (instead of opening it)
  • Cancel in the action bar exits select mode

Batch actions: Move opens the move modal with all selected items as a list (target = a directory; basenames preserved). Delete opens the confirm modal listing all items.

Disabled in multi-select: Rename (single-item only). Right-clicking a selected row drops "Rename" / "Open" from the menu.
